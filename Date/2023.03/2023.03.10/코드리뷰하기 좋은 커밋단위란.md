# 코드리뷰하기 좋은 커밋단위란?

코드리뷰(Code Review)는 소프트웨어 개발에서 중요한 단계 중 하나로, 개발자들이 작성한 코드를 다른 개발자들이 검토하고 분석하는 과정입니다.
코드리뷰를 통해 개발자는 코드의 품질을 향상시키고 버그를 예방할 수 있습니다.

코드리뷰를 수행하기 위해서는 코드가 작은 단위로 쪼개져 있어야 합니다.
이렇게 작은 단위의 코드들을 Commit 단위라고 부릅니다. 이때, 코드리뷰하기 좋은 커밋단위는 다음과 같은 특징을 갖습니다.

1. 기능 단위로 나뉘어져 있어야 합니다.

커밋 단위는 기능을 완성하는 단위로 나뉘어져 있어야 합니다.
예를 들어, 로그인 기능을 구현하고 있는 커밋 단위와 회원가입 기능을 구현하고 있는 커밋 단위는 구분되어야 합니다.
이렇게 구분되어 있는 커밋 단위를 기능 단위라고 합니다.

2. 커밋 메시지가 명확하고 간결해야 합니다.

커밋 메시지는 코드리뷰를 할 때 매우 중요합니다.
메시지가 명확하고 간결하면 코드리뷰어가 코드를 빠르게 이해할 수 있습니다. 따라서, 커밋 메시지는 기능이나 버그 수정 내용 등을 명확하게 나타내야 합니다.

3. 커밋 단위가 너무 작거나 너무 크지 않아야 합니다.

커밋 단위가 너무 작으면 코드리뷰어가 커밋을 일일이 확인하는 데 시간이 오래 걸릴 수 있습니다.
반면에 커밋 단위가 너무 크면 코드리뷰어가 커밋 전체를 분석하는 데 오랜 시간이 걸릴 수 있습니다. 적절한 커밋 단위는 기능 단위로 쪼개어진 단위입니다.

4. 커밋 단위가 독립적이어야 합니다.

커밋 단위는 서로 독립적이어야 합니다. 예를 들어, 기능을 구현하기 위해 여러 파일을 수정했을 경우, 파일 하나만 커밋하면 안 됩니다.
대신, 파일 전체를 커밋하거나, 기능 구현에 필요한 파일들만 커밋해야 합니다.
