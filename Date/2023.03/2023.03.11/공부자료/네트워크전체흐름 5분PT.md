# 5분 PT

네트워크를 설명하기에 앞서, 면접 단골질문인 검색창에 www.google.com을 입력하면 무슨 일이 일어나는지와 관련해서 네트워크를 전체적으로 설명하겠습니다.

우리가 검색이라는 행동을 수행할 때는 원하는 정보를 얻기 위해 서버까지 수많은 네트워크 노드를 거쳐가야 합니다. 이를 위해서는 중간에 IP 주소라는 것이 필요한데, 실제로 주소창에 입력하는 것은 URL 주소입니다.

주소창에 URL 주소가 입력되면 브라우저는 DNS 서버에 URL 주소를 보내서 DNS 쿼리를 통해 주소를 IP 주소로 변환합니다. 그 후, 소켓 라이브러리를 통해 HTTP 메시지를 만들고 해당 자원을 갖고 있는 서버를 검색합니다. HTTP 메시지에는 요청하는 자원의 위치 정보가 포함되어 있습니다. 이때 가장 근처에 있는 캐시 서버부터 검색합니다. 캐시 서버에는 일반적으로 자주 검색되는 자원들을 같이 모아놓는 곳입니다.

이후, HTTP 메시지가 전송 계층을 통과하면서 출발지 포트 번호와 목적지 포트 번호, 그리고 데이터가 정확하게 전달될 수 있도록 신뢰성을 보장할 수 있게 도와주는 수많은 정보를 담은 TCP 헤더에 담아 HTTP 메시지를 캡슐화합니다.

그리고 인터넷 계층을 통과하면서 DNS 서버에서 획득한 출발지 IP 주소와 응용 계층에서 HTTP 메시지를 만든 후 검색되었던 목적지 서버의 IP 주소를 IP 헤더에 담아 IP 패킷을 만듭니다. (IP 헤더에는 주소뿐만 아니라 헤더의 길이나 패킷의 길이 등 다른 여러 가지 정보들도 추가로 생성해서 담게 됩니다.)

이후 데이터 링크 계층을 통과하면서 출발지 MAC 주소(컴퓨터의 MAC 주소)와 라우터의 MAC 주소를 이더넷 프레임에 등록하여 이를 또 다시 캡슐화합니다.

여기까지의 과정을 간략하게 정리하자면 컴퓨터에서 출발해서 DNS 서버를 통해 출발지 검색 그리고 원하는 자원을 요청하면서 목적지를 알아내고 신뢰성 있게 도착하기 위해 TCP 헤더와 목적지에 정확하게 도착하기 위해 IP 주소를 담은 IP패킷이 LAN의 범위의 네트워크를 통과하면서 스위치를 통과하고 WAN의 범위인 라우터를 통과합니다. 이때 라우터의 MAC 주소가 필요합니다. 라우터는 수많은 네트워크 노드를 거쳐가야 하기 때문에 무수히 많은 라우터를 거쳐갈 것입니다.

그리고 이제 서버에 도착하기 전, 마지막 라우터에서 이더넷 프레임에 서버의 MAC 주소를 담아 캡슐화하여 데이터 링크 계층을 지나 물리 계층에서 전기 신호로 변환하여 데이터를 전달합니다. 그리고 물리 계층부터 서버의 응용 계층에 도착하기 전까지 계층을 하나씩 지나면서 이더넷 프레임의 역 캡슐화, IP 패킷의 역 캡슐화, TCP 세그먼트의 역 캡슐화를 진행해서 최종적으로 HTML 요청 메시지가 서버에 도착하게 됩니다.

서버에 만약 요청 메시지에 있는 자원이 있다면 다시 응답 메시지에 자원을 담아 클라이언트로 다시 보내게 됩니다. 이때의 과정은 앞서 설명했던 과정의 역순이라고 생각하면 되고, 클라이언트에서 서버로 올 때 거쳐왔던 라우터 경로는 서버에서 클라이언트로 돌아갈 때는 동일하지 않습니다.
